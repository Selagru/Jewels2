процесс установки (сборка):
	В терминале открытом в папке проекта набираем последовательно:
		docker-compose run web python /code/manage.py migrate --noinput
		docker-compose run web python /code/manage.py createsuperuser --username admin --email q@q.com
		docker-compose up -d --build


Запуск собранного проекта: docker-compose up


Проект имеет два эндпоинта:
	1.Загрузка файла для обработки:
		a. https://127.0.0.1:8000/upload/ - есть простенький интерфейс для загрузки файлов с выдачей результата обработки.
			Возможные результаты:
				"Файл не верного формата" - не верный тип файла.
				"Файл отсутствует" - не выбран файл.
				"Следующие строки не были обработаны из за ошибок в них:{errors}" - CSV файл обработан не полностью, строки с ошибками выводятся в теле ошибки.
				"OK - файл был обработан без ошибок" - всё классно.

		b. Запрос через Insomnia / Postman на https://127.0.0.1:8000/upload/ (доступные методы: POST)
			Content-Type: multipart/form-data
			форма: deals - "файл.csv"

	2. Выдача обработанных данных:
		https://127.0.0.1:8000/top_5_customers/ (доступные методы: GET) - показывает пятёрку покупателей, которые потратили больше всего + список камней, согласно условию.
		(данные кэшируются на 15 минут)


Так же имеется админка по адресу https://127.0.0.1:8000/admin/ в которой можно посмотреть:
	пользователей,
	сделки (с возможностью поиска по следующим полям: ['customer', 'item', 'date'],
	Камни (с возможностью поиска по названию).


Типичный процесс работы с сервисом:
	1. по адресу https://127.0.0.1:8000/upload/ - загружаем CSV файл.
	2. по адресу https://127.0.0.1:8000/top_5_customers/ - смотрим топ 5 транжир и их любимые камни.
	3. по адресу https://127.0.0.1:8000/admin/ - балуемся.
